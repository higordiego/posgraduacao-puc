## Resumo.
Problemas com histórias de usuários.
  - Histórias muito pequenas causam problemas de estimativas quando a ordem da implementação afeta consideravalmente as estimativas.
    - Os dados do relatorio devem ser salvo em arquivo .doc.
    - Os dados do relatorio devem ser salvo em arquivo .xls.
  - HIstorias dependentes.
    - Podem causa dificuldade no planejamento das iterações.
    - A inclusão de uma história em uma iteração incluir outras relacionada.
    - Ocorre quando história são pequenas ou divididas inapropriadamente.
    - Solução pode ser combinar histórias.
    - Lembrar da divisão em camadas (fatias do bolo).
  - Goldplating - (dar de bandeja) - entregar algo que o cliente não pediu.
    - Acontece quando são entregues funcionalidade que não foram solicitadas ou além do que é necessário para sua implementação.
    - Difícil impressionar o cliente com a participação constante em projetos ágeis.
    - Demonstra algo valor para documentação para documentação em detrimento da conversação.
    - Uso de cartões (espaço limitado).
  - Iniciar desenvolvimento sem visão.
    - É um erro iniciar o desenvolvimento sem nenhuma visão do produto.
    - Deve haver uma visão disponível para enunciar claramente o cliente selecionado, suas necessidades e atributos críticos.
    - Esta visão ajudar determinar quais recursos deverão ser implementados e garante criação de um produto útil.
